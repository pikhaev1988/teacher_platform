# Teacher Platform

Платформа для учителей с генерацией поурочных планов и материалов (Word + PowerPoint) на основе GigaChat.

## Стек

- Python 3.11+
- Django 6
- Redis (опционально, только если снова включать Celery)
- Docker/Docker Compose (по желанию)

## Основные возможности

- Регистрация и личный кабинет учителя.
- Создание запросов на генерацию поурочного плана.
- Синхронная генерация плана через GigaChat **без Celery/Redis**.
- Автоматическое формирование:
  - DOCX‑документа с поурочным планом;
  - PPTX‑презентации по содержанию плана.
- История генераций, скачивание ранее созданных материалов.
- Простая биллинговая модель (баланс/подписка, интеграция с YooKassa – заготовка).

## Локальный запуск

```bash
git clone https://github.com/pikhaev1988/teacher_platform.git
cd teacher_platform
python -m venv .venv
.venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

### Настройка окружения

Создайте файл `.env` в корне проекта на основе `.env.example` и заполните значения:

- `SECRET_KEY` – секретный ключ Django.
- `DEBUG` – `True` для локальной разработки.
- `ALLOWED_HOSTS` – обычно `localhost,127.0.0.1`.
- Параметры БД (по умолчанию используется SQLite, можно оставить как есть).
- `OPENAI_API_KEY` – если будете использовать OpenAI (сейчас не обязателен).
- `GIGACHAT_CLIENT_ID`, `GIGACHAT_AUTH_KEY` – данные из кабинета разработчика Сбера для GigaChat.
- Параметры YooKassa при необходимости.

### Миграции и запуск сервера

```bash
python manage.py migrate
python manage.py createsuperuser  # по желанию
python manage.py runserver
```

Приложение будет доступно по адресу `http://127.0.0.1:8000/`.

## Как работает генерация поурочного плана

Основная логика находится в приложении `apps/generator`:

- `views.create_request` – форма и обработчик создания запроса.
- `services.GPTService` – обёртка над API GigaChat (получение access token + вызов модели).
- `services.DocumentGenerator` – генерация DOCX и PPTX по тексту плана.
- `models.GenerationRequest` / `GeneratedDocument` – хранение запросов и файлов.

Процесс:

1. Пользователь заполняет тему, класс и предмет.
2. `create_request` синхронно вызывает `GPTService.generate_lesson_plan(...)`.
3. Ответ GigaChat сохраняется в БД и используется для генерации DOCX/PPTX.
4. Пользователь попадает на страницу статуса и может скачать файлы.

## Важно по безопасности

- Файл `.env` **не коммитится** в репозиторий (`.gitignore` уже настроен).
- Пример `.env.example` не содержит реальных ключей – добавьте свои локально.
